# SDE-1 Prep — Arjit & Sreehitha

Two people. One goal: Better SDE-1 offers. This repo is how we keep each other honest.

## Why this exists
Talking about prep is easy. Showing up daily is the actual game. This repo is proof — for both of us — that the work happened. No exceptions, no "I'll log it later."

## Structure

```
.
├── a_space/        → Arjit's space (C++, Go)
│   ├── dsa/        → daily DSA log
│   ├── lld/        → LLD problems, solved in C++
│   └── project/    → practice projects (optional)
│
├── s_space/        → Sreehitha's space (Java, Spring Boot)
│   ├── dsa/        → daily DSA log
│   ├── lld/        → same LLD problems, solved in Java
│   └── project/    → practice projects (optional)
│
└── common/         → shared resources, not logs
    ├── books/      → PDFs — DSA, system design, CS fundamentals books
    └── theory/     → PDFs/notes — backend, common interview questions, CS fundamentals
```

## The Rule
**Commit every day.** Even if it's one row in a table. Even if the day was bad. A missed log day is worse than a bad solve. Consistency is the whole point of this repo — speed comes later.

## How each folder works
- **`dsa/`** → no code here, code lives on LeetCode. Just log what you solved (template + example inside each `dsa/README.md`).
- **`lld/`** → actual code + design notes, one folder per problem. Same problem, picked by either of us, solved independently in our own language, tracked in our own README.
- **`project/`** → optional. Use it when you want hands-on practice instead of more grinding.
- **`common/`** → shared resources only. Put a PDF here once, mention it to the other person. No duplicating the same book/notes across both spaces.

## The deal
No ghosting the log. No "I'll catch up on Sunday." Small daily reps, tracked honestly, compound into an offer. That's the whole strategy.

We start now. We get it.
